# auth state contract (broadcast)

This layer publishes **two independent state streams**. They must never be conflated.

## Two streams, two semantics

| Stream | Type | Lifetime | Purpose | Publisher |
|---|---|---|---|---|
| `flutter-bloc` | `AuthBloc<AuthEvent, AuthState>` | per-action transient | UI feedback for the dispatcher | Flutter AuthBloc |
| `native` | `AuthStateChannel.stream` | long-lived session | session-wide state broadcast (app-global) | Native (Android/iOS) |

**Failure modes when conflated:**
- Interpreting `AuthBloc.AuthState.success` as "session active" — nothing happens after success, so the UI gets stuck in a permanent success state
- Expecting session expiry from the BLoC — the BLoC has no idea; native emits it. Without a global listener wired in, expiry goes undetected.

## flutter-bloc state definitions (per-action)

| State | Emitted when | Meaning |
|---|---|---|
| `initial` | right after BLoC creation | no dispatch yet |
| `loading` | on entry to every `_on*Requested` handler | in progress |
| `loaded(profileJson)` | only on `_onProfileRequested` success | profile JSON available |
| `success` | on success of every other handler | operation complete |
| `failure(Failure)` | on failure of any handler | carries Failure info |

## native auth state definitions (session-wide)

| State | Emitted when | Required reaction |
|---|---|---|
| `loggedOut` | no token at app start / after logout / after withdraw | block protected screens |
| `loggedIn(userId, role, exp)` | token valid + role extracted successfully | allow protected screens, branch on role |
| `refreshing` | 401 → refresh in progress | keep UI as-is, ideally hold new requests |
| `expired(message?)` | refresh failed / forced expiry | SnackBar + redirect to `/login` |

## Reactor matrix (manual + auto-generated by lint via reverse index)

| Source state | Reactor page | Reaction |
|---|---|---|
| `flutter-bloc:success` | [[ui-submit]], [[signup]], [[withdraw]], [[password-change]] | navigation |
| `native:expired` | global listener | SnackBar + `/login` |
| `native:loggedIn` | every role-gated page | role-based branching |
